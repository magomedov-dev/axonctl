# Архитектура


Эта страница для контрибьюторов и любопытных. Она описывает внутренние слои за
публичным API. Стабильный контракт — только имена, реэкспортированные из
`axonctl`; всё, что ниже, — детали реализации.

## Слои (снизу вверх)

```
PC client ──(adb forward tcp:<port> tcp:9008)──► on-device agent

fleet/      controller · watcher · ports · adb · groups · executor   (the park)
device.py   Device facade · gestures · wait · retry
tree/       geom · node · tree · window · selector   (pure, no I/O)
events/     bus · state                              (per-device)
rpc/        client · pending · ids · errors
conn/       ws · router · connection · reconnect     (transport)
```

Каждое устройство полностью независимо: своё соединение, свой генератор id, свой
реестр ожидающих запросов и своя шина событий. Единственный общий примитив —
глобальный семафор конкурентности у исполнителя.

## Транспорт (`conn/`)

- **`WsClient`** (`ws.py`) — тонкая обёртка над `websockets` за протоколом
  `WebSocketTransport` (шов, который тесты подменяют фейком). `max_size` отключён,
  потому что дампы бывают большими.
- **`FrameRouter`** (`router.py`) — классифицирует каждый входящий кадр: бинарный
  (первые 4 байта BE = id) → pending; текст + `id` → pending; текст + `event` →
  шина событий.
- **`DeviceConnection`** (`connection.py`) — по одному на устройство. Единственная
  **супервизорная** таска крутит цикл чтения и цикл ping и при обрыве переподключается
  с backoff (переоткрывая тот же URI; adb она никогда не трогает). Поток событий она
  включает при подключении (конкурентно, чтобы обрыв обнаружился, даже если агент так
  и не ответил). `close()` чисто останавливает всё.
- **`ReconnectPolicy`** (`reconnect.py`) — экспоненциальный backoff с потолком и
  джиттером.

## RPC (`rpc/`)

- **`IdGenerator`** — монотонные id, переполнение на 2³²−1, никогда не переиспользует
  ожидающий id.
- **`PendingRegistry`** — id → future. Обрабатывает двухчастный (вариант A) ответ
  скриншота (метаданные + бинарь резолвятся вместе); `cancel_all` будит всех при
  обрыве.
- **`RpcClient`** — `call`/`call_binary` с `asyncio.timeout` на каждый вызов и
  гарантированной очисткой. Ответы с ошибкой превращаются в типизированные подклассы
  `RpcError`.
- **`errors.py`** — иерархия исключений и маппинг wire-кода → класс.

## События (`events/`)

- **`EventBus`** — пофановтное распределение по устройству на короткоживущих
  подписчиков (по очереди на каждого); `interrupt()` будит ожидающих при транзиентном
  обрыве, `close()` — при постоянном.
- **`ScreenState`** — последние `screen` + `package`, питается событиями и дампами.

## UI-дерево (`tree/`, чистое)

`Bounds`/`Point`, `UiNode`/`UiTree` (парсинг + навигация; ссылки на родителей строятся
лениво), `Window`/`WindowList` и `Selector` (вычисляется на ПК). Никакого I/O —
тривиально тестируемо.

## Фасад устройства (`device.py`, `gestures.py`, `wait.py`, `retry.py`)

`Device` собирает всё вышеперечисленное в обращённый к пользователю API. `WaitEngine`
выполняет событийные ожидания; `GestureBuilder` собирает параметры жестов;
`RetryPolicy` обрабатывает `STALE`.

## Парк (`fleet/`)

- **`AdbWatcher`** — оборачивает `adbutils.track_devices()` (блокирующий итератор,
  переброшенный из демон-потока) в асинхронный поток attach/detach.
- **`AdbBridge`** — операции adb (`forward`, `shell`, `launch`, ...), вынесенные в
  потоки; **`PortAllocator`** раздаёт локальные порты.
- **`TagIndex` / `DeviceGroup`** — тег → серийники и резолвинг цели.
- **`FleetController`** — реестр и жизненный цикл: attach → port → forward →
  connect → register; detach → close (остановить реконнект) → удалить forward →
  освободить порт. Координирует реконнект и detach, чтобы они никогда не гонялись.
- **`FleetExecutor`** — выполняет сценарий на каждом устройстве под общим глобальным
  семафором, собирая `Outcome` по каждому устройству в `Results`.

## Инварианты проектирования

- **Никогда не блокировать event loop.** Блокирующий adb / парсинг идут через
  потоки/пулы.
- **Таймаут на каждом внешнем await.** Ни один вызов или ожидание не висит вечно.
- **Чистая отмена.** `close()/stop()` отменяют и дожидаются каждую таску.
- **Изоляция по устройствам.** На горячем пути нет глобальных локов; падение одного
  устройства никогда не топит весь парк.
- **Событийно, а не поллингом.** Ожидания реагируют на `screenChanged`.

## Тестирование

- **unit** — чистая логика (дерево/селекторы, сборка жестов, переполнение id, маппинг
  ошибок, классификация кадров, порты, группы), плюс ASCII-страж на ключи протокола.
- **integration** — внутрипроцессный фейковый агент (настоящий WebSocket-сервер)
  прогоняет весь путь: ping/dump, событийные ожидания, корреляция скриншотов,
  реконнект, attach/detach парка и исполнитель.

Формат провода см. в [Протоколе](PROTOCOL.md).
