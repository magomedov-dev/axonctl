# Ожидания


Ожидания — это критичное к скорости ядро axonctl. Они **событийны и никогда не
поллят**: ожидание подписывается на поток `screenChanged` агента, делает базовый
дамп и передампливает **только** когда экран реально изменился.

## Методы ожидания

```python
node = await device.wait_for(Selector.id("com.app:id/ok"), timeout=10)   # -> UiNode
await device.wait_gone(Selector.id("com.app:id/spinner"), timeout=15)    # -> None
await device.wait_package("com.android.settings", timeout=10)            # -> None
toast = await device.wait_toast(timeout=5)                               # -> str
```

- [`wait_for`][axonctl.Device.wait_for] резолвится, когда селектор совпал;
  возвращает узел.
- [`wait_gone`][axonctl.Device.wait_gone] резолвится, когда селектор не совпадает
  ни с чем.
- [`wait_package`][axonctl.Device.wait_package] резолвится, когда пакет на
  переднем плане равен заданному значению. (Агент не отдаёт имена активити —
  передавай **пакет**, например `"com.android.settings"`. `wait_activity` —
  сохранённый алиас.)
- [`wait_toast`][axonctl.Device.wait_toast] возвращает текст следующего тоста.

Все бросают [`WaitTimeout`][axonctl.WaitTimeout] по истечении срока и
[`ConnectionLost`][axonctl.ConnectionLost], если связь оборвалась посреди
ожидания.

## Почему не просто цикл по `find`?

`find` в цикле поллит: он передампливает по таймеру независимо от того, изменилось
что-то или нет, тратя ресурсы и добавляя задержку. Событийное ожидание
передампливает только при реальном изменении, поэтому оно и дешевле, и быстрее
реагирует. Предпочитай `wait_for` ручным циклам с ретраями.

## Поток событий включается автоматически

Библиотека включает поток событий агента **при подключении устройства** (и заново
после реконнекта) — `setEventStream` вызывать самому не нужно. Поскольку он включён
с самого начала, ожидания работают сразу, а тосты захватываются ещё до того, как ты
их запросишь.

## Тосты эфемерны — гонка обработана

Тосты всплывают мгновенно (без дебаунса) и исчезают за доли секунды. Классический
паттерн — «сделать что-то, затем прочитать фидбэк»:

```python
await device.click(Selector.text("Sign in"))
feedback = await device.wait_toast(timeout=5)   # e.g. "Wrong password"
```

Если тост всплыл в зазоре между возвратом `click` и подпиской `wait_toast`, наивная
реализация его пропустила бы. axonctl буферизует последний тост с меткой времени,
поэтому `wait_toast` возвращает тост, всплывший в коротком окне *перед* вызовом
(настраивается через `lookback`).

## Транзиентное «нет активного окна»

При запуске приложений дамп может на мгновение упасть с
`ACCESSIBILITY_DISABLED` (окна на переднем плане ещё нет). Ожидания терпят это —
они продолжают ждать и перепроверяют на следующем событии, а не падают.

## Таймауты

Каждое ожидание принимает `timeout` (в секундах) и бросает `WaitTimeout`, если
условие не выполнено вовремя:

```python
from axonctl import WaitTimeout

try:
    await device.wait_for(Selector.id("com.app:id/home"), timeout=8)
except WaitTimeout:
    ...  # handle the slow/stuck case
```
